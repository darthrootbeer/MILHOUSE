::context
Milhouse already writes a log, but the operator experience needs more live “heartbeat” in the terminal: when a task starts, show a single clean line describing what it’s working on (pulled from the TODO selection / task file).

::done-when
- On every iteration start, Milhouse prints a single “Starting work:” line to the terminal that includes the current task one-liner (preferably including the TODO id like `[ABC123]` when available)
- The same “Starting work:” line is also written to `.milhouse/out.txt` (human-readable)
- Output is readable in 80x24 terminals and doesn’t spam (max: a few key lines per iteration)
- Works with and without `gum` (same info; nicer styling when gum is present)

::steps
1. Choose a “source of truth” for the one-liner (prefer `MILHOUSE_TASK.md` checkbox lines created from `TODO.md` selection)
2. Add a small helper in `scripts/milhouse.sh` that extracts the first unchecked checkbox line and normalizes it to a single line (trim, collapse whitespace)
3. At iteration start (before `cursor-agent` runs), print the “Starting work:” one-liner using `show_info` (gum) or ANSI/plain fallback
4. Also log the same one-liner via `log_out` so `tail -f .milhouse/out.txt` always shows it
5. Add one more “Finished iteration:” summary line with the same task one-liner + progress counts to keep the terminal story coherent

::avoid
- Do not print the whole TODO list or the whole task file
- Do not print per-tool-call chatter (keep it to iteration-level milestones)
- Do not change task-selection behavior beyond what’s needed to surface the one-liner

::notes
- If multiple checkboxes exist, “first unchecked” is the simplest and usually correct
- If no unchecked checkbox exists, show “All criteria complete” (avoid confusing blanks)
