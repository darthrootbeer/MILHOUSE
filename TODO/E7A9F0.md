::context
While Milhouse runs, the operator experience should feel premium: a single, stylish dashboard (or at least a fancier on-screen log) that makes progress and state obvious, with color and structure — closer to “Claude Code terminal mode” than a scrolling wall of text.

::done-when
- While Milhouse runs, there is a dashboard-style screen option that shows:
  - current mode + workspace + model
  - iteration number + progress (done/total/remaining)
  - context health (emoji or labeled status)
  - last N notable events (iteration start, agent exit code, rotation/gutter, etc.)
- Dashboard refreshes without flooding scrollback (redraws in-place)
- Ctrl+C exits cleanly and leaves the terminal usable
- Works without gum (fallback: keep existing simple output; still structured and readable)

::steps
1. Decide the minimal dashboard “data sources” (reuse `MILHOUSE_TASK.md`, `.milhouse/iteration`, and `.milhouse/out.txt`)
2. Add a new UI mode (e.g. `MILHOUSE_UI=dashboard` or `--dashboard`) that:
   - clears/redraws the screen on an interval
   - renders a header, a progress block, and a “recent events” block
3. In gum mode, use `gum style` blocks for a consistent look (borders, padding, muted separators)
4. In non-gum mode, use ANSI + box-drawing characters (already used elsewhere) with width-safe truncation
5. Ensure the dashboard never blocks agent execution; it should be an optional viewer layer, not the core log sink

::avoid
- Do not build a complex multi-process TUI framework
- Do not remove `.milhouse/out.txt` (it remains the durable audit trail)
- Do not rely on non-portable terminal features; must work in typical macOS terminals

::notes
- “Sexy” here means: clear hierarchy, intentional spacing, stable layout, and minimal noise
- Keep the default experience simple; make dashboard opt-in at first to avoid surprises
