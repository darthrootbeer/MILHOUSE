::context
Implement commit count-based rotation: count commits made in current iteration, rotate if >10 commits. Track commits before/after iteration.

::done-when
- Function count_commits_in_iteration() returns commit count
- Tracks commit count before iteration starts
- Tracks commit count after iteration ends
- Calculates difference
- Function should_rotate_commits(commit_count) returns true if >10

::steps
1. Before iteration: COMMIT_COUNT_START=$(git rev-list --count HEAD)
2. After iteration: COMMIT_COUNT_END=$(git rev-list --count HEAD)
3. Calculate: COMMIT_DIFF=$((COMMIT_COUNT_END - COMMIT_COUNT_START))
4. Add should_rotate_commits(count): return true if count > 10
5. Add to main loop: check after iteration, rotate if true

::avoid
- Don't parse commit messages (just count)
- Don't track commit authors/content
- Keep threshold configurable but default 10

::notes
Commit-based rotation: if >10 commits in one iteration, rotate. Many commits suggests lots of changes, context filling. Uses git rev-list --count. Default threshold 10 commits.
